# Wallet Service

Приложение на Spring Boot для управления кошельками с поддержкой конкурентных операций.

---

## Описание

Проект предоставляет REST API для работы с кошельками:

- **POST /api/v1/wallet** – выполнить операцию с кошельком (DEPOSIT или WITHDRAW)
- **GET /api/v1/wallets/{walletId}** – получить текущий баланс кошелька

Особенности:

- Потокобезопасная обработка операций с использованием транзакций и блокировки записи (`PESSIMISTIC_WRITE`)
- Обработка ошибок: несуществующий кошелёк, недостаточно средств, некорректные данные
- Конфигурация через переменные окружения
- База данных PostgreSQL с миграциями Liquibase
- Полностью покрыто тестами, включая стресс-тест для конкурентных операций

---

## Технологии

- Java 17
- Spring Boot 3
- Spring Data JPA
- PostgreSQL
- Liquibase
- Docker / Docker Compose
- JUnit 5, Testcontainers

---

## Запуск

```bash
docker compose up --build
```

Приложение будет доступно по адресу `http://localhost:8080`.

---

## Конфигурация

Все параметры задаются в файле `.env` и применяются без пересборки контейнеров:

| Переменная             | Описание                        |
|------------------------|---------------------------------|
| `DB_NAME`              | Имя базы данных                 |
| `DB_USER`              | Пользователь БД                 |
| `DB_PASSWORD`          | Пароль БД                       |
| `DB_PORT`              | Порт БД (хост)                  |
| `DB_MAX_CONNECTIONS`   | max_connections PostgreSQL       |
| `SERVER_PORT`          | Порт приложения                 |
| `HIKARI_MAX_POOL_SIZE` | Размер пула соединений HikariCP |
| `HIKARI_MIN_IDLE`      | Минимум idle соединений         |

Значения по умолчанию находятся в `.env`. Для изменения достаточно отредактировать файл и перезапустить контейнеры:

```bash
docker compose up -d
```
